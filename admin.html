💳 Stripe Card / Google Pay
Accept international card payments via a Stripe Payment Link.
Setup: Stripe Dashboard → Payment Links → Create link → copy URL below. Get Publishable Key from Developers → API Keys.
🔴 eWAY AUS / NZ
Accept payments via eWAY hosted payment page (Australia & New Zealand).
Setup: eWAY MyAccount → API Keys → copy API Key. Hosted Payments → create page → copy URL.
📲 Fonepay QR Nepal
Display your Fonepay merchant QR — customer scans and pays instantly.
Setup: Get Merchant Code from your Fonepay merchant account. Upload your QR image to Cloudinary and paste the URL below.
🟢 eSewa Nepal
Nepal's most popular digital wallet. Use EPAYTEST merchant code for sandbox testing.
Setup: eSewa merchant portal → Merchant Code (scd). Use EPAYTEST for sandbox. Set redirect URLs to your published site.
🟣 Khalti Nepal
Accept payments via Khalti digital wallet. Supports live and test keys.
Setup: Khalti merchant dashboard → API Keys → copy your Public Key (starts with live_public_key_ or test_public_key_).

Backend Configuration
💾 Using Local Storage — data is stored in this browser only.
🗄 Data Storage Backend

Enable a cloud backend so orders and enquiries sync across all computers. Local Storage is always the automatic fallback if a cloud backend is not set up or temporarily fails.
Only one cloud backend should be active at a time. Toggle it on, enter credentials, then click ⚡ Test Connection before saving.

💾 Local Storage Always On — Fallback
Browser-only. Zero setup, works offline. Data does not sync between computers or browsers — each device sees only its own data.
🔥 Firebase Firestore Off
Google's real-time cloud database. Orders and enquiries sync instantly across all devices and browsers. Recommended.
How to set up Firebase (free):
  1. Go to console.firebase.google.com → Create project (or use existing)
  2. Firestore Database → Create database → choose Test mode (you can tighten rules later)
  3. Project Settings ⚙ → Your apps → Add Web app → Copy the config values below
⚡ Supabase Off
Open-source Firebase alternative backed by PostgreSQL. Good for developers comfortable with SQL databases.
How to set up Supabase (free):
  1. Go to supabase.com → New project
  2. SQL Editor → create tables: orders, enquiries, admins (each with an id text primary key)
  3. Project Settings → API → copy Project URL and anon key below
👤 User Authentication

Allow customers to create accounts, sign in with email or Google, save favourites, and track orders. Uses Firebase Authentication — requires Firebase to be enabled above.

Show Sign In / Sign Up Button Off
Displays the Sign In button in the navigation bar. Turn OFF to completely hide user accounts from visitors — site works as a public catalogue.
One-time setup (5 min) — Firebase Console:
  1. Open console.firebase.google.com → your project
  2. Left sidebar → Authentication → Get started
  3. Sign-in method tab → enable Email/Password → Save
  4. To use Google Sign-In → also enable Google provider → add your support email → Save
  5. Authentication → Settings → Authorized domains → add your GitHub Pages domain (e.g. yourname.github.io)

New accounts must click a verification link before they are fully active. Recommended for real customers.

Customers can choose "Continue as Guest" at checkout instead of creating an account. Recommended ON to maximise sales.

Google Sign-In Off
Allow users to sign up / sign in with their Google account — one click, no password needed. No extra API keys required; Firebase handles the entire OAuth flow.
Enable Google provider in Firebase (already done if you followed step 4 above):
  1. Firebase Console → Authentication → Sign-in method
  2. Click Google → toggle to Enable
  3. Enter a Project support email (your Gmail) → Save
  4. No Client ID or Client Secret needed — Firebase manages OAuth tokens automatically

☁ Media Storage

☁ Cloudinary Off
Free image hosting with automatic optimisation, transformations and a CDN. Recommended for most sites.
How to set up Cloudinary (free):
  1. Sign up at cloudinary.com → Dashboard
  2. Copy your Cloud Name from the dashboard
  3. Settings → Upload → Add upload preset → set to Unsigned, copy its name
🪣 Amazon S3 Off
Store images in your own AWS S3 bucket. Full control, suits enterprise setups with existing AWS infrastructure.
How to set up AWS S3:
  1. AWS Console → S3 → Create bucket (enable public access for images)
  2. IAM → Create user → Attach AmazonS3FullAccess → Create access key
  3. S3 bucket → Permissions → CORS: allow PUT from your domain
